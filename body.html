12 Column Grids!

This component is full-width, but the content is constrained by the "container".

Notice how the green background stretches across the screen, but the content is still aligned with the rest of the page? This is because we are using a container that changes width depending on the viewport (screen) width!

Device Widths

Extra Small

Since 2015 more than half of Google searches were via mobile devices, so mobile designs need to be accessible and easy to navigate. This is a fluid layout, meaning there is not a fixed width container.

  • Usually single column
  • Only hamburger navigation
  • Simplified content (no carousels, background videos etc)

Small

The portrait tablet view is tricky as you have to find a balance between the "mobile feel" of a website and adding more content to the now fixed-width layout, meaning there is a margin either side of the content.

  • Up to two columns
  • Hamburger navigation preferred
  • Simplified content (no carousels, background videos etc)

Medium

If the user isn't on a mobile, they are probably using a device with this screen width. Known as the normal "desktop" view, this is more common among laptops and landscape tablets. Confused yet?

  • Usually two or three columns
  • Full navigation
  • Full content, no holds barred

Large

The big Kahuna. The Big Lewbowski. Big Daddy Box Meal. While the size is bigger, the container is still a fixed width of 1170px. This means that even if this was on super big monitor, the actual content size won't change, just the width of the margin. The content and layout usually doesn't differ from the "Medium" size.

The Container

The container is used to keep elements within a fixed width layout. The table above shows how wide the container is for each device in the "Fixed" column.

If the viewport / screen width is larger than the fixed width a margin is given to each side equally, keeping the container in the centre of the page.

The 15px Gutter

All elements in Bootstrap have a 15px horizontal gutter either side of them, and most have a 15px horizontal padding (inside-margin).

Even though elements in the first and last columns still have the 15px gutter either side, the row that contains them has a margin of -15px on either side. This means that the elements still appear to be aligned with the container.
